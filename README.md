# Maxson Plant GIS Transpiler

A zero-footprint, environment-aware utility for converting ESRI Shapefiles to GeoJSON. This is designed for high-fidelity mapping of municipal assets (wastewater treatment, piping, and electrical) and integration into Three.js digital twin pipelines.

## Features
- **Environment Decoupling**: Uses `.env` files for site-specific coordinates and paths.
- **Lean Implementation**: Pure Shell/Bash script—no heavy Python or JavaScript dependencies.

## Use Case at Maxson
- **ArcGIS to Open**: SHP files generated by the City of Memphis GIS Mapping team were converted to GEOJSON, for modular use and enhanced leveragability. 
- **Bulk Processing**: Automatically iterates through assets like `PipeMain`, `WaterMain`, and `Electrical`.
- **DCS Alignment**: Ideal for mapping Ovation DCS database points to spatial geometries.

## Prerequisites
- **GDAL/ogr2ogr**: The core engine for this tool.
  ```bash
  sudo apt install gdal-bin
  ```

## Setup
1. Clone the repository and navigate to the root.
2. Create a `.env` file (see `.env.bak` for the template):
   ```bash
   SHP_SOURCE_DIR="../../Documents/gis/ShapeFiles_4_8_26"
   GEOJSON_OUT_DIR="../../Documents/gis/ShapeFiles_4_8_26/geojson/"
   ORIGIN_LATITUDE=35.0456
   ORIGIN_LONGITUDE=-90.0489
   ```

## Usage
### 1. Load the Environment
```bash
[ -f .env ] && export $(grep -v '^#' .env | xargs) || echo "No .env found"
```

### 2. Run the Conversion
```bash
chmod +x ./scripts/shp-to-geojson.sh
./scripts/shp-to-geojson.sh
```

## Manual Conversion (One-off)
If you need to process a single file outside of the bulk workflow, call `ogr2ogr` directly:
```bash
ogr2ogr -f "GeoJSON" output.json input.shp -t_srs "EPSG:4326" -lco COORDINATE_PRECISION=7
```
This is how the `shp-to-geojson.sh` script works, with environment values routed to control bulk conversion for the input folder, the output folder, the origin latitude, and the origin longitude.
